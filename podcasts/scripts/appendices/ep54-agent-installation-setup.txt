Now let’s talk about what goes wrong.

[ALEX]
For GitHub Copilot in VS Code, one issue is that agents do not appear after typing the at sign. Common causes are missing or outdated extensions. The fix is to reinstall GitHub Copilot and GitHub Copilot Chat from the VS Code Marketplace.

Another issue is a “Cannot find agents” error. A common cause is that the dot github slash agents slash folder is missing. The remedy is to clone the full accessibility-agents repository rather than copying files manually.

A third problem is chat not working because Copilot is not signed in. In that case, sign in again through the Copilot interface.

[JAMIE]
And the source specifically calls out cache and restart steps too.

[ALEX]
Right. If VS Code seems stuck in a bad state, delete the VS Code cache and restart VS Code. Those two actions can clear extension-loading issues.

[JAMIE]
What about the other platforms, even if the source doesn’t spell out every row in the table we only partially saw?

[ALEX]
We can still teach the main compatibility mindset from the appendix. For Claude Code, likely failure points include npm-based installation problems, missing Node.js, or incomplete authentication. Check the CLI version, confirm Node 18 or later, and rerun auth if needed.

For Gemini CLI, likely issues include a missing or invalid API key, environment variables not being set in the current shell, or Node.js being too old. Recheck gemini config show, verify your key, and confirm node double dash version.

For Claude Desktop, the big risks are malformed dot m c p dot json, incorrect file paths, or forgetting to fully restart the application after configuration changes. Validate the JSON carefully and review logs.

For Codex CLI, common trouble areas are incomplete initialization, Node or npm path problems, or misunderstanding the roles configuration. Rerun codex init if necessary and verify codex role list works.

[ALEX]
Yes. Across all platforms: verify versions first, confirm authentication, confirm network access, make sure the repository is complete, prefer the latest stable tools, and restart the affected app or shell after major changes. If a command behaves oddly, test whether your shell can actually find it with a version command before assuming the agents are at fault.
