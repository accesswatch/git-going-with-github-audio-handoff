Before we close, can we give learners a compact mental model for the whole lesson?

[ALEX]
Absolutely. Here it is.

Local work happens in your clone. You make edits, save files, stage intentionally, and commit with a clear message. Your branch is local until you push. Once pushed, GitHub can compare that branch against main and create a pull request. The pull request is where collaboration becomes visible: description, diffs, comments, review verdicts, and merge decisions. Accessibility matters at every layer, from using the Command Palette and Source Control panel accessibly, to reading diffs with F7, to writing review comments that clearly identify blocking issues versus suggestions.
