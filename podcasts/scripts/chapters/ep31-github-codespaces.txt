Let's talk maintenance. Once you've created a Codespace, what does it mean to manage it?

[ALEX]
Three common actions are stopping it, deleting it, and resuming it.

Stopping a Codespace means shutting down the running cloud machine without necessarily destroying the environment. That's useful when you're done for now and want to preserve the workspace while not continuing to use compute time.

Deleting a Codespace removes that Codespace environment entirely. That's a much bigger action.

Resuming a Codespace means reopening one you've used before and continuing where you left off.

[JAMIE]
