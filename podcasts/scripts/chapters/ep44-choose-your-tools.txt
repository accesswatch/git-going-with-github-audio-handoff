[JAMIE]

If you’re using github dot dev, open the repository with the period key or the github dot dev URL, make a small edit in the browser editor, inspect the changed files, and explore the commit interface.

If you’re using VS Code and it’s already set up, open the local folder or cloned repository, edit a file, then use Source Control or the terminal to inspect changes and prepare a commit.

If you’re using GitHub Desktop, open or clone the repository there, edit the file in your preferred editor, return to GitHub Desktop to review the diff, add a commit message, and get ready to commit and sync.

If you’re using GitHub CLI, navigate to the repository locally, make a small change in your editor of choice, use Git commands and gh where appropriate to inspect status or connect the work back to GitHub, and practice the command-line flow.
